OpaqueAction
An action with implementation-specific semantics.
The semantics of the action are determined by the implementation. OpaqueAction is introduced for implementation-specific actions or for use as a temporary placeholder before some other action is chosen. Modelio specific: The action may be filled in one language only. Code generation MDA components may allow specific note types in order to allow the user to fill code in their generated language.
