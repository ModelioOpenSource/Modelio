Managing Modelio modules

Some functionnalities are brought by specific modules (Code generation, documentation, etc.). Modules are addons that can be plugged to Modelio to extends its capabilities.
Example: to generate Java code from a UML model, you need to deploy the Java Designer module.

Where can I get Modelio modules?
Modelio modules can be downloaded from the Modelio online store.

How do I make the modules available to Modelio?
The downloaded modules have to be stored in the Modelio Modules Catalog.

The Modelio Modules Catalog

Steps

  1. Run the "Configuration / Modules catalog..." command
  2. In the Modules catalog window, click on "Add a module to the catalog..." and use the file browser to select the modules (*.jmdac files)
  3. Click on "Close" button when your catalog is up-to-date

Deploying a module in the current project
To deploy a module from the catalog in the project, use the Project configuration window:

Key:

  1. List of modules available to the project
  2. List of modules contained in the modules catalog
  3. Expand/Shrink catalog view button
  4. Selected module properties

Steps:

  1. Expand the Catalog view by clicking on the button
  2. Select a module in the catalog
  3. Click on the button to add the module in the project modules list
  4. The project's modules can be activated and deactivated by checking/unchecking their tickbox
  5. Modules parameters can be edited in the Project's modules parameters view
  6. A module can be removed from the Project's modules list. To do so, select the module and click on the button
